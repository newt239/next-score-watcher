# アタック25形式仕様書

## 概要

アタック25形式は、Score Watcherで利用できるパネル獲得型の競技クイズルールです。テレビ番組「パネルクイズ アタック25」で実際に使用されている形式を再現しており、4人のプレイヤーが5×5＝25枚のパネルをオセロ形式で奪い合います。

他の形式と異なり、**スコアが盤面の空間状態（どのプレイヤーがどのパネルを保持しているか）から導出される**点が最大の特徴です。

## 基本仕様

### ゲームルール名

- **システム名**: `attack25`
- **表示名**: アタック25
- **短縮説明**: 5×5の25パネルをオセロ形式で奪い合う形式です。
- **詳細説明**: 4人のプレイヤーが、正解するたびに5×5＝25枚のパネルを1枚ずつ獲得します。獲得したパネルと自分の既存パネルで相手のパネルを挟むと、オセロのように挟まれたパネルが自分の色に反転します。すべてのパネルが埋まったとき、最も多くのパネルを獲得しているプレイヤーが優勝です。

### ゲーム設定項目

| 項目                    | 設定値の例 | 説明                                                     |
| ----------------------- | ---------- | -------------------------------------------------------- |
| `options.attack_chance` | `true`     | アタックチャンスを有効にするかどうか（デフォルト: 有効） |

**重要な制約**：

- プレイヤー数は**必ず4人固定**です。4人以外ではゲームを開始できません。
- 盤面は5×5＝25パネル固定です。

### プレイヤーの色

プレイヤーは登録順に以下の色が割り当てられます。

| 並び順      | 色  |
| ----------- | --- |
| プレイヤー1 | 赤  |
| プレイヤー2 | 緑  |
| プレイヤー3 | 白  |
| プレイヤー4 | 青  |

白パネルは数字が見えづらくならないよう、文字色を黒で表示します。

## ゲームルール

### パネルの獲得

- 各問、正解者が空きパネルを1枚指定し、自分の色で点灯させます。
- **オセロ式反転**: 指定したパネルと自分の既存パネルで相手のパネルを挟むと、8方向それぞれで挟まれた相手パネルが自分の色に反転します。空きマスや盤端で連鎖は途切れます。

### パネルの選択制約

「はさめる所があれば、必ずはさめるパネルをとる」というルールに従い、獲得できるマスは状況に応じて制限されます。

1. **反転できるマスがある場合**: 反転（挟み）が発生するマスのみ選択可能。
2. **反転できるマスがない場合**: 点灯済みパネルに隣接（8方向）する空きマスのみ選択可能。
3. **盤面が空（初手）の場合**: すべての空きマスが選択可能。

選択できないマスはボタンが無効化され、クリックできません。

### アタックチャンス（オプション）

`options.attack_chance` が有効な場合のみ発生します。

- 残りの空きパネルが**5枚以下**になった状態での次の正解者がアタックチャンスの対象です。
- 対象者は通常通りパネルを獲得した後、点灯済みパネルを**1枚消す（空きに戻す）**ことができます。
- 消去は任意で、スキップも可能です。
- アタックチャンスはゲーム中**1回のみ**発生します。

### ゲーム終了

- 全25パネルが埋まったらゲーム終了です。
- 最も多くのパネルを保持しているプレイヤーが優勝（`win`）、それ以外は敗北（`lose`）になります。

#### タイブレーク（同数時の順位決定）

最多保持プレイヤーが複数いる場合、以下の優先順位で順位を決定します。

1. **保持パネル数**（多い方が上位）
2. **正解数**（多い方が上位）
3. **誤答数**（少ない方が上位）

3項目すべてが同じ場合は同順位となります。

## スコア計算ロジック

### 初期状態

各プレイヤーの初期状態：

- **スコア**: 0（保持パネル数）
- **正解数**: 0
- **誤答数**: 0
- **状態**: `playing`（プレイ中）

### スコアの算出

アタック25形式では、ログを時系列に畳み込んで盤面を再構築し、その盤面から各プレイヤーの保持パネル数を集計してスコアとします。

1. `variant === "correct"` かつ `panel` を持つログを順に処理する。
2. 各ログで `applyReversiFlip` を適用し、オセロ反転を反映する。
3. ログに `removed_panel` があれば、そのパネルを空きに戻す（アタックチャンスの消去）。
4. 最終盤面の各プレイヤーの保持パネル数が `score` になる。

### 正解時の処理

- 指定パネルを獲得し、オセロ反転を盤面に反映。
- 正解数を1増加。
- アタックチャンス対象ターンの場合は、`removed_panel` 付きのログとして記録される場合がある。

### 誤答時の処理

- 誤答数を1増加。
- 盤面には影響しないが、操作履歴・元に戻す機能の一貫性のためログに記録される（`panel` なし）。

### 状態管理

- **`playing`**: ゲーム続行中（盤面に空きパネルが残っている）
- **`win`**: 全パネル充填時の最多保持プレイヤー
- **`lose`**: 全パネル充填時の最多保持以外のプレイヤー

## データ管理

アタック25形式は、AQL形式と同様に既存のログ畳み込みアーキテクチャに乗っています。

- 「正解者がどのパネルを選んだか」を記録するため、`LogDBProps` に任意フィールド `panel`（獲得パネル番号 0-24）と `removed_panel`（アタックチャンスで消したパネル番号 0-24）を追加しています。
- これらは非インデックスの任意プロパティのため、Dexieのスキーマ変更（バージョン更新）は不要です。
- ゲーム状態はすべてIndexedDBの `logs` テーブルで管理され、元に戻す／やり直し・マルチプロファイルにそのまま対応します。

## UI表示・操作方法

### スコアボード表示

アタック25形式では専用の表示レイアウトが使用されます：

- 問題文のすぐ下に「次に何をすればよいか」を案内するメッセージを横幅いっぱいで表示。
- 4人のプレイヤーカード（色・名前・保持パネル数）と、5×5の25パネルグリッドを表示。
- デスクトップではプレイヤーカードとパネルグリッドを横並びで表示。

### 操作方法

基本的な操作方法については、[共通操作方法](../common-operations.md)を参照してください。

#### アタック25形式固有の操作（プレイヤー先選択方式）

1. **正解者のプレイヤーカードをクリック**: 正解者を選択。
2. **獲得するパネルをクリック**: 選択可能なパネルのみクリックでき、獲得とオセロ反転が行われる。
   - アタックチャンス対象の場合は、続けて消すパネルを選択するモードに移行。
3. **消去モード**: 点灯済みパネルをクリックで消去、または「消さずに確定」ボタンでスキップ。
4. **誤答ボタン**: プレイヤー選択中に押すと誤答として記録（盤面には影響しない）。

#### キーボード操作の制約

アタック25形式では、パネルの指定が必要なため、数字キーによる正誤入力は無効化されています。「一つ戻す」「スルー」のキー操作は利用できます。

## 実装詳細

### 関連ファイル

- **盤面ロジック**: `src/utils/attack25.ts`
- **スコア計算ロジック**: `src/utils/computeScore/attack25.ts`
- **ゲーム設定定義**: `src/utils/rules.ts` (194-204行目)
- **型定義**: `src/utils/types.ts`
- **専用UIコンポーネント**: `src/app/(board)/games/[game_id]/board/_components/Attack25/Attack25.tsx`
- **テストファイル**: `src/utils/computeScore/__tests__/attack25.test.ts`

### 主要な関数

`src/utils/attack25.ts` に盤面計算ロジックを集約し、スコア計算とUIコンポーネントの両方から再利用しています。

- `applyReversiFlip(board, index, playerId)`: 指定マスに配置し、8方向で挟んだ相手パネルを反転した新しい盤面を返す。
- `getClaimablePanels(board, playerId)`: プレイヤーが獲得できるマスの一覧を返す（反転マス優先、無ければ隣接マス、初手は全マス）。
- `computeAttack25Board(logs, attackChanceEnabled)`: ログを畳み込み、盤面とアタックチャンス使用状況を計算する。
- `isAttackChanceActive(board, attackChanceUsed, attackChanceEnabled)`: 次の正解がアタックチャンス対象かを判定する。
- `countPanels(board)`: 盤面から各プレイヤーの保持パネル数を集計する。

### 計算アルゴリズム

```typescript
// ログを時系列に畳み込んで盤面を再構築
for (const log of logs) {
  if (log.variant !== "correct" || log.panel === undefined) continue;
  // オセロ反転を適用
  board = applyReversiFlip(board, log.panel, log.player_id);
  // アタックチャンスでの消去を反映
  if (log.removed_panel !== undefined) {
    board[log.removed_panel] = null;
  }
}

// 保持パネル数をスコアに設定
const counts = countPanels(board);
```

## 使用場面

アタック25形式は以下のような場面で活用できます：

- 「パネルクイズ アタック25」形式の大会・練習
- 早押し技術に加えてパネルの陣取り戦略が求められる競技
- オセロの反転を活かした逆転要素のある競技設計

## 注意事項・制約

### 必須制約

- **プレイヤー数**: 必ず4人で実施（4人以外ではゲームを開始できない）

### 運用上の注意

- パネルの選択は「はさめる所があれば必ずはさめるパネルをとる」というルールに従って制限されます。意図したマスが選択できない場合は、より優先度の高いマスがあるか確認してください。
- アタックチャンスはゲーム中1回のみ発生します。事前にプレイヤーへの説明を推奨します。

## 戦略的要素

アタック25形式特有の戦略的要素：

1. **オセロ反転を狙った獲得**: 相手のパネルを大量に挟んで一気に反転させる
2. **角・端の確保**: 反転されにくい角や端のパネルを優先的に狙う
3. **アタックチャンスの活用**: 終盤の特別ターンで相手の角パネルを消去する

これらの要素により、早押し技術だけでなく、盤面を見極めた陣取り戦略が求められる形式となっています。
